web sockets

what/why/how/where

what

  • a computer communications protocol
  • full-duplex channels over a single TCP connection
  • tl;dr a two way connection between a client and server

why

  • real-time communication!
  • no need to refresh to get updates
  • better than polling (depending on the use-case of course)

how

  1. client sends http request to server to open connection
  2. server agrees, response: 
    101 Switching Protocols
  3. handshake is created, TCP/IP connection is left open
  4. bi-directional messages go back and forth
